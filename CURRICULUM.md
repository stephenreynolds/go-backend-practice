# Go Backend Curriculum

## Phase 1: Go Fundamentals (Week 1-2)

### Week 1: Language Basics
- Day 1: Hello Go — setup, packages, main, fmt
- Day 2: Variables, types, constants, zero values
- Day 3: Functions, multiple returns, named returns
- Day 4: Control flow — if, for, switch
- Day 5: Arrays, slices, and slice operations

**Weekly Project:** CLI tool — simple task manager (add, list, complete)

### Week 2: Go Building Blocks
- Day 6: Maps and structs
- Day 7: Pointers and memory
- Day 8: Methods and receivers
- Day 9: Interfaces and polymorphism
- Day 10: Error handling patterns

**Weekly Project:** File-based data store with CRUD operations

## Phase 2: Concurrency & I/O (Week 3-4)

### Week 3: Goroutines & Channels
- Day 11: Goroutines basics
- Day 12: Channels — unbuffered and buffered
- Day 13: Select statement and timeouts
- Day 14: sync package — WaitGroup, Mutex
- Day 15: Context and cancellation

**Weekly Project:** Concurrent web scraper

### Week 4: Working with Data
- Day 16: JSON encoding/decoding
- Day 17: Working with files
- Day 18: HTTP client basics
- Day 19: SQL databases (PostgreSQL)
- Day 20: Database migrations

**Weekly Project:** REST API with database persistence

## Phase 3: Web Services (Week 5-6)

### Week 5: HTTP Servers
- Day 21: net/http basics
- Day 22: Routing and handlers
- Day 23: Middleware patterns
- Day 24: Request validation
- Day 25: Authentication (JWT)

**Weekly Project:** User authentication service

### Week 6: API Design
- Day 26: RESTful design principles
- Day 27: OpenAPI/Swagger documentation
- Day 28: Rate limiting
- Day 29: Caching strategies
- Day 30: API versioning

**Weekly Project:** Full REST API with docs and rate limiting

## Phase 4: Production Ready (Week 7-8)

### Week 7: Testing & Quality
- Day 31: Unit testing basics
- Day 32: Table-driven tests
- Day 33: Mocking and interfaces
- Day 34: Integration testing
- Day 35: Benchmarking

**Weekly Project:** Add comprehensive tests to previous project

### Week 8: Deployment
- Day 36: Docker basics for Go
- Day 37: Multi-stage builds
- Day 38: Configuration management
- Day 39: Logging and observability
- Day 40: Graceful shutdown

**Weekly Project:** Dockerized, production-ready service

## Phase 5: Advanced Patterns (Week 9-10)

### Week 9: Architecture
- Day 41: Project structure patterns
- Day 42: Dependency injection
- Day 43: Repository pattern
- Day 44: Clean architecture
- Day 45: Event-driven patterns

**Weekly Project:** Refactor to clean architecture

### Week 10: Scaling
- Day 46: gRPC basics
- Day 47: Message queues (Redis/RabbitMQ)
- Day 48: Distributed tracing
- Day 49: Health checks and readiness
- Day 50: Kubernetes basics

**Weekly Project:** Microservice with gRPC and message queue
